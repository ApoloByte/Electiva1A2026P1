# electiva-1

## Que es Git?

- Es un sistema que permite rastrear cambios en el tiempo dentro de un proyecto.

## Que es github?

- GitHub es una plataforma basada en la nube diseñada para almacenar, gestionar y colaborar en proyectos de desarrollo de software

## Relacion entre Git y Github?

- Ambos permiten trabajar en equipo y llevar un control de versiones, pero Git es la herramienta que maneja el versionado y GitHub es el servicio que aloja los repositorios Git en línea.

## Que es el .gitignore?

- Es un archivo de texto que le dice a Git qué archivos o directorios debe ignorar en un proyecto.

## Comandos Git

- git branch: Listar rama.
- git checkout-: Verificar rama.
- git switch: Cambiar de rama.
- git remote add origin: Agregar un repositorio remoto.
- git remote set origin: Cambiar la URL del repositorio remoto.
- git clone: Clonar un repositorio remoto.
- git add: Agregar archivos al área de preparación.
- git status: Verificar el estado de los archivos.
- git log: Ver el historial de commits.
- git merge: Combinar ramas.
- git stash: Guardar cambios sin hacer commit.
- git stash pop: Recuperar y eliminar el último stash.
- git pull origin: Traer cambios del remoto y hacer merge.
- git push origin: Subir cambios al remoto.
- git diff: Ver cambios no agregados al staging.
- git reset —soft: Deshacer último commit sin perder cambios (quedan en staging).
- git commit —amend: Modificar el último commit (mensaje o contenido).
